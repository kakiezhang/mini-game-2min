# 《120秒下班》MVP 技术实现方案 v0.1

## 1. 技术目标

本方案服务于 MVP v0.1，目标不是一次性做成完整商业版本，而是快速做出可试玩、可调数、可扩展到微信小游戏的核心闭环。

核心目标：

| 目标 | 说明 |
|---|---|
| 快速可玩 | 优先在 PC Web 跑通完整 120 秒单局 |
| 移动优先 | 画面、UI、输入按竖屏手机设计 |
| 配置驱动 | 玩家、敌人、技能、道具、时间线、地图点位尽量配置化 |
| 模块清晰 | 输入、战斗、技能、道具、UI、结算边界明确 |
| 可移植 | 玩法逻辑不强绑定 DOM 或浏览器事件，便于后续适配微信小游戏 |

---

## 2. 推荐技术栈

### 2.1 MVP 推荐方案

```text
TypeScript
Vite
Phaser 3
Canvas/WebGL 渲染
JSON/TS 配置表
```

选择原因：

| 技术 | 作用 | 原因 |
|---|---|---|
| TypeScript | 玩法逻辑开发 | 类型约束能降低配置和实体字段错误 |
| Vite | Web 开发与构建 | 启动快，适合快速调试 |
| Phaser 3 | 2D 游戏框架 | 内置场景、相机、输入、碰撞、动画和资源管理 |
| Canvas/WebGL | 渲染层 | 适合 2D 俯视角割草游戏 |
| TS 配置表 | 数值配置 | 开发期类型更清晰，后续可转 JSON |

### 2.2 微信小游戏适配策略

MVP 第一阶段先完成 PC Web 可试玩版本。微信小游戏适配作为后续平台层任务处理。

适配原则：

```text
玩法逻辑不直接读取 window/document
输入模块统一输出 InputState
音频、存档、生命周期放到 PlatformAdapter
资源路径集中管理
画布尺寸由统一的 ScaleManager 控制
```

如果后续要求直接以微信小游戏为主发布，需要在 M8/M10 阶段评估：

| 方案 | 适用情况 | 代价 |
|---|---|---|
| Phaser Web 构建 + 微信适配层 | 已经完成 Web MVP，希望低成本迁移 | 需要处理适配器、包体、触屏和音频限制 |
| Cocos Creator 重建项目 | 微信小游戏发布优先级高，后续内容量大 | 初期开发和工程管理成本更高 |

当前 MVP 建议先采用 Phaser Web 方案，尽快验证玩法闭环。

## 4. 核心架构

### 4.1 游戏状态

使用明确的单局状态机管理流程：

```text
Ready
  ↓
Playing
  ↓
LevelUpPaused
  ↓
Playing
  ↓
Success / Failed
  ↓
Restart
```

状态定义：

| 状态 | 说明 |
|---|---|
| Ready | 场景加载完成，等待开局 |
| Playing | 正常游戏中，倒计时、敌人、技能全部更新 |
| LevelUpPaused | 升级选择中，玩法暂停，UI 可交互 |
| Success | 成功撤离结算 |
| Failed | 死亡或超时失败结算 |

### 4.2 每帧更新顺序

每帧按固定顺序更新，减少系统之间互相踩状态。

```text
读取输入
  ↓
更新时间线
  ↓
更新玩家移动与状态
  ↓
刷新和更新敌人
  ↓
更新技能与投射物
  ↓
处理碰撞和伤害
  ↓
处理道具拾取
  ↓
处理经验和升级
  ↓
处理撤离读条
  ↓
检查成功/失败
  ↓
刷新 UI
```

`LevelUpPaused` 状态下只更新升级 UI，不更新倒计时、敌人、技能和碰撞。

## 5. 输入系统

### 5.1 统一输入结构

键盘、虚拟摇杆和技能按钮统一转换成同一个输入对象。

```ts
export type InputState = {
  moveX: number;
  moveY: number;
  dashPressed: boolean;
  pointerActive: boolean;
};
```

约束：

| 字段 | 说明 |
|---|---|
| moveX / moveY | 范围 -1 到 1，内部做归一化 |
| dashPressed | 只表示本帧是否触发主动技能 |
| pointerActive | 手机摇杆是否正在拖动 |

### 5.2 PC 输入

| 按键 | 输出 |
|---|---|
| W / ↑ | moveY = -1 |
| S / ↓ | moveY = 1 |
| A / ← | moveX = -1 |
| D / → | moveX = 1 |
| Space | dashPressed = true |

### 5.3 手机输入

| 控件 | 输出 |
|---|---|
| 左下角虚拟摇杆 | moveX / moveY |
| 右下角技能按钮 | dashPressed |

虚拟摇杆只影响输入状态，不直接移动玩家。

## 6. 地图与坐标

### 6.1 坐标系统

采用世界坐标和屏幕坐标分离：

| 坐标 | 用途 |
|---|---|
| 世界坐标 | 玩家、敌人、道具、房间、碰撞区域 |
| 屏幕坐标 | UI、摇杆、按钮、提示 |

摄像机跟随玩家移动，UI 固定在屏幕坐标。

### 6.2 地图配置

地图先用矩形区域配置，不依赖复杂 TileMap。

```ts
export type MapArea = {
  id: string;
  name: string;
  x: number;
  y: number;
  width: number;
  height: number;
};
```

关键点位：

```text
playerSpawn
accessCardSpawn
coffeeSpawn
bossSpawn
elevatorZone
```

### 6.3 障碍物

MVP 第一版只使用少量矩形障碍：

```text
办公桌
会议桌
咖啡机
老板桌
电梯门装饰
```

碰撞策略：

| 对象 | 策略 |
|---|---|
| 玩家 vs 障碍 | 阻挡 |
| 敌人 vs 障碍 | MVP 可先弱化，允许绕行不足时滑动修正 |
| 投射物 vs 障碍 | MVP 可先穿透障碍 |

## 8. 敌人与刷怪

### 8.1 刷怪阶段

刷怪由时间线控制，不直接写死在敌人模块中。

| 时间 | 刷怪内容 |
|---|---|
| 0 秒 | Bug 怪开始刷新 |
| 15 秒 | 需求变更怪加入刷新池 |
| 40 秒 | 会议怪加入刷新池 |
| 90 秒 | 老板出现 |

### 8.2 刷怪位置

普通敌人从屏幕外或地图边缘生成，优先保证玩家能看到敌人涌入。

规则：

```text
根据玩家当前位置选择视野外方向
生成点限制在地图边界内
避免生成在玩家半径过近区域
```

### 8.3 敌人行为

MVP 使用直接追击：

```text
计算玩家方向
归一化方向
按速度靠近玩家
接触后造成伤害
```

会议怪额外效果：

```text
接触玩家后施加 30% 减速，持续 1 秒
```

老板额外效果：

```text
第 90 秒出现
持续追击玩家
接触高伤害
P1 阶段周期性召唤需求变更怪
```

## 9. 技能系统

### 9.1 技能原则

技能系统分为技能数据、技能等级、技能运行逻辑。

```text
技能是否拥有
  ↓
当前技能等级
  ↓
读取等级配置
  ↓
按技能类型执行效果
```

技能升级只修改等级，不直接改散落在各处的运行参数。

### 9.2 键盘飞镖

实现方式：

```text
根据等级配置生成 N 个 orbit hitbox
围绕玩家按角速度旋转
与敌人碰撞造成伤害
每个敌人设置短暂受击间隔
```

配置字段：

```text
count
damage
radius
angularSpeed
hitCooldown
```

### 9.3 鼠标连点器

实现方式：

```text
按冷却时间寻找最近敌人
向目标方向发射投射物
投射物命中敌人造成伤害
根据穿透数继续飞行或销毁
```

配置字段：

```text
cooldown
damage
projectileSpeed
pierce
projectileCount
```

### 9.4 咖啡冲刺

实现方式：

```text
读取当前移动方向
如果没有移动方向，则使用最近一次有效方向
短时间提升位移速度或直接位移一段距离
触发冷却
```

配置字段：

```text
cooldown
distance
duration
knockback
postSpeedBoost
charges
```

### 9.5 离职信清屏

实现方式：

```text
按冷却时间在玩家周围释放圆形冲击波
命中范围内敌人造成伤害
老板只承受较低倍率伤害
```

配置字段：

```text
cooldown
radius
damage
waveCount
bossDamageMultiplier
```

## 10. 道具系统

### 10.1 门禁卡

门禁卡由时间事件在第 35 秒生成。

拾取后：

```text
player.hasAccessCard = true
隐藏门禁卡实体
更新 HUD
切换目标箭头逻辑
```

### 10.2 咖啡

咖啡在茶水间固定刷新。

拾取后：

```text
回复 20 血
施加 20% 移速加成，持续 3 秒
进入刷新冷却
```

### 10.3 离职信

离职信可以从敌人小概率掉落，或先在固定点刷新。

拾取后：

```text
立即释放一次范围伤害
普通敌人高伤害
老板低倍率伤害
```

## 11. 时间事件系统

时间事件统一配置：

```ts
export type TimelineEvent = {
  time: number;
  id: string;
  once: boolean;
  action: TimelineAction;
};
```

关键事件：

| 时间 | 事件 |
|---|---|
| 0 | 开局提示，Bug 怪开始刷新 |
| 15 | 需求变更怪加入刷新 |
| 35 | 生成门禁卡 |
| 40 | 会议怪加入刷新 |
| 70 | 提示电梯即将开放 |
| 80 | 电梯开放 |
| 90 | 老板出现 |
| 120 | 未撤离则失败 |

时间暂停规则：

| 状态 | 倒计时是否推进 |
|---|---|
| Playing | 是 |
| LevelUpPaused | 否 |
| Success / Failed | 否 |

## 12. 撤离系统

撤离条件：

```text
玩家持有门禁卡
电梯已开放
玩家在电梯区域内
玩家未死亡
```

读条规则：

```text
进入电梯区域后开始累计
离开区域后清零
累计达到 2 秒触发成功结算
读条期间仍然会受到伤害
```

失败优先级：

```text
如果读条期间死亡，进入死亡失败
如果时间归零且未成功，进入超时失败
如果同一帧读条完成，成功优先于超时
```

## 13. UI 实现

### 13.1 HUD

基础 HUD：

| UI | 位置 | 数据来源 |
|---|---|---|
| 血条 | 顶部左侧 | PlayerState |
| 倒计时 | 顶部中间 | TimelineSystem |
| 门禁卡状态 | 顶部右侧 | PlayerState.hasAccessCard |
| 经验条 | 顶部下方 | PlayerState.exp / expToNext |
| 技能图标 | 底部或右侧 | SkillSystem |
| 目标箭头 | 屏幕边缘 | ObjectiveSystem |
| 撤离读条 | 玩家上方或屏幕中部 | EvacuationSystem |

### 13.2 提示消息

提示使用轻量 toast 队列：

```text
同一时间最多显示 1 条主提示
新提示可覆盖旧提示
提示持续 1.5-2.5 秒
关键提示可带轻微缩放动画
```

### 13.3 升级面板

升级时：

```text
暂停玩法系统
显示 3 个技能强化选项
点击或键盘选择其中 1 个
应用升级
关闭面板
恢复游戏
```

## 14. 目标箭头

目标优先级：

```text
门禁卡已出现且未拾取：指向门禁卡
已拾取门禁卡且电梯已开放：指向电梯
已拾取门禁卡但电梯未开放：可弱提示电梯位置或不显示
```

实现方式：

```text
计算目标世界坐标到屏幕中心方向
如果目标在屏幕内，箭头可隐藏或指向目标上方
如果目标在屏幕外，箭头固定在屏幕边缘
```

## 16. 性能策略

MVP 阶段重点控制敌人和投射物数量。

策略：

| 项目 | 策略 |
|---|---|
| 敌人数量 | 设置全局上限 |
| 投射物数量 | 对象池复用 |
| 伤害检测 | 先用半径碰撞，必要时做空间分桶 |
| UI 更新 | 数据变化时更新，避免每帧重建 DOM |
| 特效 | MVP 使用简单图形和短生命周期粒子 |

建议上限：

```text
普通敌人：80-120
投射物：80
掉落道具：20
浮动伤害数字：30
```

## 19. 风险与处理

| 风险 | 表现 | 处理 |
|---|---|---|
| 技能太多导致闭环变慢 | 一直在补技能，撤离流程迟迟不可玩 | 先完成键盘飞镖，其他技能后补 |
| 刷怪过多导致卡顿 | 90 秒后帧率下降 | 敌人数量上限、对象池、简化碰撞 |
| 手机操作卡手 | 摇杆遮挡或角色被障碍卡住 | 减少障碍，扩大通道，优化摇杆区域 |
| 目标不清晰 | 玩家不知道去哪里 | 时间提示 + 目标箭头 + 状态 UI |
| 微信适配返工 | Web 逻辑依赖浏览器 API | 平台能力统一封装到 PlatformAdapter |

## 20. 首轮开发落点

```text

竖屏画布
办公室区域地图
玩家移动
Bug 怪刷新和追击
键盘飞镖自动攻击
经验升级
门禁卡 35 秒刷新
电梯 80 秒开放
老板 90 秒出现
120 秒成功/失败结算
基础 HUD
PC 键盘 + 手机摇杆
```

首轮完成后再补：

```text
鼠标连点器
咖啡冲刺完整等级
离职信清屏
咖啡补给
会议怪减速
老板召唤
数值调优
视觉和音效反馈
```
