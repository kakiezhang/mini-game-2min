# 《120秒下班》MVP 开发任务列表 v0.1

## 1. 拆分原则

MVP 开发优先保证一局游戏可以完整跑通，再逐步补齐技能、敌人、道具、UI 和移动端体验。

任务优先级分为：

| 优先级 | 说明 |
|---|---|
| P0 | 可玩闭环必须项，缺失则不能验收 MVP |
| P1 | MVP 体验必须项，缺失会明显影响节奏或手感 |
| P2 | 第一版增强项，可在闭环稳定后补齐 |

开发顺序建议：

```text
项目基础
  ↓
核心闭环原型
  ↓
战斗与成长
  ↓
任务目标与撤离
  ↓
UI 与提示
  ↓
移动端适配
  ↓
验收调优
```

## 3. MVP 最小开发路径

如果要最快做出可试玩版本，建议先完成以下任务：

```text
T001-T008
T010-T018
T019-T023
T029-T039
T040-T045
T055-T064
T065-T069
T081-T087
```

这条路径会先形成完整闭环：

```text
进入游戏
  ↓
移动
  ↓
打怪
  ↓
升级
  ↓
拿门禁卡
  ↓
等电梯
  ↓
躲老板
  ↓
撤离成功 / 死亡失败 / 超时失败
```

鼠标连点器、完整咖啡冲刺、离职信、咖啡补给、老板召唤和更细的数值调优，可以在闭环可玩后补齐。

## 4. 模块责任边界

| 模块 | 责任 | 不负责 |
|---|---|---|
| 玩家模块 | 玩家属性、移动、受伤、死亡 | 输入来源、UI 展示 |
| 输入模块 | 键盘、摇杆、技能按钮，输出统一输入状态 | 玩家具体移动逻辑 |
| 敌人模块 | 敌人生成、行为、受击、死亡 | 技能选择、升级 UI |
| 技能模块 | 自动攻击、主动技能、技能等级效果 | 敌人刷新节奏 |
| 道具模块 | 道具生成、拾取、效果触发 | 长期角色养成 |
| 地图模块 | 地图布局、障碍、关键点位 | 时间事件逻辑 |
| 时间事件模块 | 关键时间点触发 | UI 样式 |
| 升级模块 | 经验、升级、3 选 1 | 具体技能渲染 |
| 撤离模块 | 电梯状态、撤离条件、读条 | 门禁卡生成 |
| UI 模块 | HUD、提示、箭头、弹窗 | 游戏规则判定 |
| 结算模块 | 成功、失败、重开 | 单局内战斗逻辑 |
| 配置模块 | 数值和点位配置 | 业务规则解释 |
| 适配模块 | 竖屏、触屏、尺寸适配 | 具体玩法数值 |
